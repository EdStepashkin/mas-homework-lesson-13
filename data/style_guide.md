# NovaTech Solutions — Style Guide

## Tone of Voice

NovaTech спілкується **впевнено, дружньо та експертно**. Ми — технічна компанія, але говоримо простою мовою. Наш голос — це голос досвідченого колеги, який пояснює складні речі зрозуміло.

### Характеристики тону:
- **Впевнений**: ми знаємо свій продукт і ринок. Без "можливо", "напевно", "ми думаємо".
- **Дружній**: ми на "ти" з читачем. Без формальностей та бюрократизму.
- **Експертний**: ми підкріплюємо тези фактами та прикладами. Не "просто повірте".
- **Практичний**: кожна стаття має давати actionable value. Не абстрактна теорія, а конкретні кроки.

## Заборонені формулювання

❌ НЕ використовувати:
- "Революційний" / "Інноваційний" / "Проривний" — банальні маркетингові кліше
- "Лідер ринку" / "Найкращий у своєму класі" — самопроголошення без доказів
- "Просто" / "Легко" — знецінює зусилля читача
- "Синергія" / "Парадигма" / "Дисрапт" — корпоративний жаргон
- Пасивний стан ("було зроблено") — завжди активний стан ("ми зробили")
- Довгі речення (>25 слів) — розбивайте на короткі

## Рекомендовані формулювання

✅ Використовувати:
- "Дозволяє" замість "революціонізує"
- "За нашим досвідом" замість "безперечно"
- "Конкретний приклад: ..." — завжди ілюструйте тези
- "Крок 1, Крок 2, ..." — структуровані інструкції
- Дані та числа: "на 40% швидше", "3 з 5 команд"
- Запитання до читача: "Як ви зараз ведете бекліг?"

## Правила для різних каналів

### Blog (довга форма, 800-1500 слів)
- Заголовок H1 — інтригуючий, з ключовим словом
- Вступ — 2-3 речення, hook + обіцянка value
- Основна частина — H2/H3 підзаголовки, списки, приклади
- Висновок — 2-3 речення, call to action
- Тон: експертний, з прикладами

### LinkedIn (коротка форма, 150-300 слів)
- Перший рядок — hook (emoji дозволені, макс 2)
- Основна частина — 3-5 ключових тез
- Фінал — запитання або CTA
- Тон: професійний, але дружній
- Хештеги: 3-5 релевантних

### Twitter/X (мікроформа, до 280 символів)
- Одна думка = один твіт
- Thread для розгорнутих тем (5-10 твітів)
- Тон: стислий, гострий, з цифрами

## Візуальний стиль тексту

- **Emoji**: помірно, 1-2 на абзац, лише змістовні (✅, 📊, 🔧, 💡)
- **Bold**: для ключових термінів та важливих тез
- **Списки**: буллетовані для переліків, нумеровані для кроків
- **Код**: inline `code` для технічних термінів, code blocks для прикладів
- **Посилання**: до релевантних зовнішніх джерел, наших інших постів
